# 02-learning · 学习工作区（过程笔记，按节）

> 双层笔记的**过程层**：按学习顺序、按领域记每节笔记。
> 另一层**资产层**在 `03-knowledge-base/concepts/`（提炼后的稳定知识点，可 grep 复习）。

## 节级笔记怎么写
- 学完一节，在对应领域目录建 `<NN>-<节名>.md`（NN = 节号），如 `01-foundation/01-价值传递系统.md`。
- 模板：

```
# 第 N 节 · <节名>
> 学完日 / 掌握度 / 关联错题

## 要点
- 

## 易混 / 易错
- 

## 关联
- 错题：05-mistakes/<领域>.md
- 提炼资产：03-knowledge-base/concepts/<主题>.md（学完模块后由 Claude 提炼）
```

## 何时提炼进资产层
- 学完一个模块（如模块 1 框架与原则 4 节全完），Claude 把本目录的节级笔记提炼成 `03-knowledge-base/concepts/<主题>.md`。
- 提炼后节级笔记保留（过程记录），资产层用于复习时 grep 自己总结的话。

## 领域目录
- `01-foundation` 项目管理基础（12 原则、运行环境、PM 角色、生命周期）
- `02-people` 人员（团队、领导力、冲突、干系人、沟通）
- `03-process` 过程（整合/范围/进度/成本/质量/资源/风险/采购/变更 + 过程类绩效域）
- `04-business` 业务环境（合规、价值交付、治理、商业对齐）
- `05-agile` 敏捷（宣言、Scrum、Kanban、servant leadership）
- `06-mock-review` 模考复盘
